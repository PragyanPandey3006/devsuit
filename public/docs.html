Installation

Quick Start (Recommended)

The easiest way to use DevSuit is through our hosted version:

  1. Visit devsuit.software
  2. Click "Continue with GitHub"
  3. Authorize the application
  4. Start exploring your repositories!

Self-Hosted Installation

For developers who want to run DevSuit locally:

# Clone the repository
git clone https://github.com/pragyanpandey/github-devsuite.git
cd github-devsuite

# Install dependencies
npm install

# Set up environment variables
cp .env.example .env
# Edit .env with your GitHub OAuth credentials

# Start the development server
npm run dev

Note: You'll need to create a GitHub OAuth app and configure environment variables. See the Authentication section for details.

Authentication

GitHub OAuth Setup

DevSuit uses GitHub OAuth for secure authentication. Here's how to set it up:

  1. Go to GitHub Developer Settings
  2. Click "New OAuth App"
  3. Fill in the application details:
    • Application name: DevSuit
    • Homepage URL: https://your-domain.com
    • Authorization callback URL: https://your-domain.com/auth/callback
  4. Copy the Client ID and Client Secret
  5. Add them to your environment variables

Required Permissions

DevSuit requests these minimal permissions:

  • repo - Access to repository data and statistics
  • user - Read user profile information
  • read:org - Read organization membership

RepoAnalyzer - Health Checker

RepoAnalyzer provides automated health scoring and actionable recommendations for repositories.

Health Checks

README.md Present

Checks for comprehensive documentation

Verifies proper licensing information

Recent Activity

Monitors repository maintenance frequency

Scoring System

Repositories receive a score from 0-100 based on best practices:

  • • 80-100: Excellent - Well maintained repository
  • • 60-79: Good - Some areas for improvement
  • • 0-59: Needs Improvement - Several issues to address

AutoLabeler - Smart Issue Labeling

AutoLabeler uses intelligent content analysis to automatically suggest and apply labels to issues and pull requests.

How It Works

  1. Analyzes issue/PR title and description
  2. Matches content against predefined rules
  3. Suggests appropriate labels with confidence scores
  4. Optionally applies labels automatically via webhooks

Default Labeling Rules

bug - error, crash, broken
enhancement - feature, improvement
documentation - docs, readme, guide
performance - slow, memory, cpu
security - vulnerability, cve
needs-triage - default fallback

Webhook Integration

Set up webhooks for real-time automation and notifications.

Setting Up Webhooks

  1. Go to your repository settings
  2. Navigate to "Webhooks" section
  3. Click "Add webhook"
  4. Enter the webhook URL: https://your-domain.com/api/webhook/github
  5. Select "application/json" as content type
  6. Choose events: Issues, Pull requests
  7. Add webhook secret (optional but recommended)

Security Note

Always use webhook secrets to verify that requests are coming from GitHub. DevSuit automatically validates webhook signatures for security.

Troubleshooting

Common Issues

Authentication Failed

If you're having trouble signing in:

  • Clear your browser cache and cookies
  • Try signing in from an incognito/private window
  • Check if your GitHub account has the required permissions
  • Verify the OAuth app is properly configured

Rate Limit Exceeded

GitHub API has rate limits. If you hit them:

  • Wait for the rate limit to reset (usually 1 hour)
  • Use authenticated requests (higher rate limits)
  • Reduce the frequency of API calls

Repository Not Found

If you can't access a repository:

  • Check if the repository name is correct
  • Verify you have access to private repositories
  • Ensure the repository exists and isn't deleted

Frequently Asked Questions

Is DevSuit free to use?

Yes, DevSuit is completely free and open source. You can use the hosted version or deploy your own instance.

What data does DevSuit store?

DevSuit only stores your GitHub access token locally in your browser. We don't store any repository data or personal information on our servers.

Can I use DevSuit with private repositories?

Yes, DevSuit works with both public and private repositories. You'll need to grant the appropriate permissions during OAuth authentication.

How do I contribute to DevSuit?

DevSuit is open source! Visit our GitHub repository to contribute code, report issues, or suggest features.
